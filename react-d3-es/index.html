Qué es D3.js?

  • Data-Driven Documents
  • Librería de visualización de datos de bajo nivel y uso general
  • Manipula documentos basados en datos
  • Estándares de Web Abierta (SVG, HTML, CSS y Canvas)
  • Permite interacciones con tus gráficas
  • El nombre D3 viene de "data-driven documents"
  • con una API inmensa
  • y centrada en transformaciones matemáticas de esos mismos datos
  • Based on web standards as SVG, HTML, CSS and canvas
  • Una de sus más celebradas características es su integración de animaciones, y cómo permite crear interacciones que cautivan
  • Es una caja de herramientas para crear visualizaciones interactivas en la web. No es una librería de gráficas.
  • Vamos a revisar algunos proyectos basados en D3

Cómo funciona?

  • Carga datos
  • Combina los datos con elementos del DOM
  • Transforma esos elementos
  • Transiciona entre estados
  • D3 carga datos y los incorpora en el DOM.
  • Combina esos datos con elementos del DOM
  • y transforma esos elementos, transicionando entre estados si es necesario.
  • Vamos a ver mi visualización favorita
  • T: Los datos se quedan en el DOM, son más estables allí según Mike Bostock.

D3 Data Joins

  • Voy a hablar brevemente sobre los "Data Joins" de D3
  • Esto será útil para establecer similitudes respecto a la forma en que React funciona

    • Transforma el DOM seleccionando elementos y uniéndolos con datos

  • Es una estrategia de gestión del DOM
  • Con jQuery accedíamos a elements
  • Con Backbone y los MV* renderizamos vistas
  • Con React ya somos más eficientes cuando renderizamos, utilizando el Virtual DOM
  • y con D3, tenemos los Data Joins

    • Data Join

  • Imagínense que tenemos cinco letras en el DOM
  • Las seleccionamos, y añadimos datos para crear el Data Join
  • Tres grupos, las letras que se quedan (update)
  • las que necesitan ser creadas (enter)
  • las que se tienen que ir (exit)
  • esta información se representa con un objeto con tres arrays
  • Es un algoritmo de differenciación, pero en vez de comparar elementos ya renderizados, comparamos datos
  • Los datos están en el DOM. Vamos a comparar esto con el uso de keys en React

    • Patrón Update, Enter
    y Exit

    Recursos
  • vamos a ver la relación entre seleción de D3 y los datos, y luego trabajado con ellos en los tres grupos: enter, update y exit
  • Los datos enlazados a elementos existentes forman el grupo "update"
  • Los datos que faltan forman el grupo "enter" (a la izquierda)
  • el resto de datos forman el grupo "exit" (a la derecha), que representan los elementos que deben ser eliminados

import D3Line from './D3Line'

class Line extends React.Component {

    static propTypes = {...}

    componentDidMount() {
        // D3 Code to create the chart
        this._chart = D3Line.create(
            this._rootNode,
            this.props.data,
            this.props.config
        );
    }

    componentDidUpdate() {
        // D3 Code to update the chart
        D3Line.update(
           this._rootNode,
           this.props.data,
           this.props.config,
           this._chart
        );
    }

    componentWillUnmount() {
        D3Line.destroy(this._rootNode);
    }

    _setRef(componentNode) {
        this._rootNode = componentNode;
    }

    render() {
        <div className="line-container" ref={this._setRef.bind(this)} />
    }
}

class Line extends React.Component {

    static propTypes = {...}

    drawLine() {
        let xScale = d3.scaleTime()
            .domain(d3.extent(this.props.data, ({date}) => date));
            .rangeRound([0, this.props.width]);

        let yScale = d3.scaleLinear()
            .domain(d3.extent(this.props.data, ({value}) => value))
            .rangeRound([this.props.height, 0]);

        let line = d3.line()
            .x((d) => xScale(d.date))
            .y((d) => yScale(d.value));

        return (
            <path
                className="line"
                d={line(this.props.data)}
            />
        );
    }

    render() {
        <svg
           className="line-container"
           width={this.props.width}
           height={this.props.height}
        >
           {this.drawLine()}
        </svg>
    }
}
